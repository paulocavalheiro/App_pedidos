## Stack tecnológica

O frontend do App Pedidos é uma aplicação [Next.js](https://nextjs.org/) com as seguintes tecnologias:

| Tecnologia      | Versão   | Uso principal                                      |
| --------------- | -------- | --------------------------------------------------- |
| **Next.js**     | 14.1.0   | Framework React (SSR, rotas, API route opcional)   |
| **React**      | 18.2.0   | Interface e componentes                            |
| **TypeScript**  | 5.3.x    | Tipagem estática                                   |
| **Material-UI (MUI)** | 5.15.x | Componentes de UI (botões, inputs, ícones, etc.) |
| **Axios**       | 1.6.x    | Cliente HTTP para consumir a API do backend         |
| **Formik**      | 2.4.x    | Formulários (cadastro de prato, etc.)              |
| **Yup**         | 1.3.x    | Validação de schemas nos formulários               |
| **React Query** | 3.39.x   | Cache e estado de requisições à API                |

## Estrutura do projeto

- **pages/** — Rotas da aplicação (Next.js file-based routing):
  - `index.tsx` — Home (identificação do cliente: nome, CPF, mesa)
  - `prato/` — Cardápio e criação de prato
  - `pedidos/` — Lista de pedidos e tela Meu Pedido
- **services/** — Cliente Axios (`api.tsx`) e tipos compartilhados (`types.ts`)
- **Layout_component/** — Layout, Navbar, Footer, Card
- **hooks/** e **pages/.../hooks/** — Hooks React para formulários e chamadas à API
- **styles/** — Módulos CSS por página/componente
- **mui_component/** — Componentes reutilizáveis baseados em MUI (ex.: SnackBarAlert)

## Scripts principais

Na pasta `frontend`:

```bash
npm run dev      # Desenvolvimento (porta 3001 por padrão)
npm run build    # Build de produção
npm run start    # Iniciar em modo produção
```

## Porta e API

O frontend roda em **porta 3001** por padrão (`next dev -p 3001`). A URL base da API é configurada em `frontend/services/api.tsx` (ex.: `http://127.0.0.1:3000` para o backend NestJS). Ajuste essa URL ou use variáveis de ambiente se o backend estiver em outro host/porta.
