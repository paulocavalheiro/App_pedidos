## Visão geral

A API do App Pedidos é uma API REST documentada com OpenAPI 3.1. Ela expõe recursos para **categorias**, **pratos** e **pedidos**, consumidos pelo frontend Next.js.

## Base URL

Em desenvolvimento, o backend roda em:

- **URL base:** `http://localhost:3000` (ou a porta configurada no NestJS)

O frontend usa essa URL em `frontend/services/api.tsx` para todas as requisições (listar categorias, pratos, criar pedido, etc.).

## Recursos

| Recurso    | Prefixo             | Descrição                                      |
| ---------- | ------------------- | ---------------------------------------------- |
| Categoria  | `/api/categoria`    | Listar, buscar por ID e criar categorias      |
| Prato      | `/api/prato`        | Listar, visualizar, buscar por IDs, criar e atualizar pratos |
| Pedido     | `/api/pedido`       | Buscar por CPF, listar, criar e atualizar status |

## Autenticação

Atualmente a API não exige autenticação; os endpoints são abertos para uso em ambiente de desenvolvimento e no fluxo do restaurante (cliente faz pedido pelo frontend).

## Testando a API

Use a collection Postman na raiz do projeto: `backendRestaurante.postman_collection.json`. Importe no Postman para testar GET, POST e PUT sem usar o frontend.
